# Business Plan Tool — Note tecniche di progetto
**Versione 1.1 | Studio Commerciale | 2025**

---

## 1. Panoramica del progetto

Il Business Plan Tool è un'applicazione per uso interno allo studio commerciale, progettata per elaborare dati di bilancio storici e produrre prospetti contabili previsionali pluriennali. L'applicazione gira interamente nel browser aprendo un singolo file `index.html` — non richiede installazioni, server locali o dipendenze esterne.

**Obiettivo principale:** partire da dati storici di Stato Patrimoniale e Conto Economico (schema civilistico art. 2424/2425 c.c.) e, applicando driver e parametri previsionali, generare prospetti futuri coerenti e quadrati per un numero variabile di anni (massimo 8).

**Distribuzione:** i file del tool risiedono su un server Windows della rete locale dello studio. I client accedono aprendo `index.html` nel browser (Chrome o Edge) — nessun software da installare sui client. I file di progetto vengono salvati come JSON scaricabili sul PC locale dell'utente.

## 3. Modello dati

### 3.1 Schema SP e CE — struttura fissa + conti liberi

Lo schema normativo definisce i livelli fissi (mastri e sottomastri) secondo gli articoli 2424 e 2425 del codice civile. I conti foglia sono creati liberamente dall'utente per ogni progetto.

- **Mastri:** fissi, non modificabili (es. B. Immobilizzazioni, A. Valore della produzione)
- **Sottomastri:** predefiniti, editabili nel nome, aggiungibili se mancanti
- **Conti foglia:** creati dall'utente con nome libero, specifici per ogni società
- **Template condiviso:** raccoglie i conti usati nei vari progetti e li propone come suggerimento nei lavori successivi

Tutti i mastri e sottomastri sono sempre visibili nell'interfaccia anche se privi di conti valorizzati, per guidare l'inserimento nel posto corretto.

### 3.2 Struttura file di progetto JSON

```json
{
  "meta": {
    "cliente": "Rossi Srl",
    "anno_base": 2024,
    "anni_previsione": [2025, 2026, 2027],
    "scenario": "sp_ce",
    "modalita_inserimento": "rapida",
    "creato": "2025-03-15",
    "modificato": "2025-03-28",
    "stato": "in_lavorazione"
  },
  "storico": {
    "2024": { "sp": { ... }, "ce": { ... } }
  },
  "eventi": [
    { "tipo": "mutuo", "descrizione": "Mutuo Banca X", ... },
    { "tipo": "investimento", "descrizione": "Linea produzione", ... },
    { "tipo": "variazione_ricavi", ... }
  ],
  "driver": {
```

**Campo `scenario`:** valori possibili `sp_ce` | `sp_only` | `costituenda`
**Campo `modalita_inserimento`:** valori possibili `rapida` | `analitica`

## 4. Motore di calcolo

### 4.1 Granularità e aggregazione

Il motore lavora internamente a granularità **mensile** (12 mesi × N anni) e aggrega i risultati per anno per i prospetti principali. Il dettaglio mensile rimane consultabile tramite drill-down.

### 4.2 Sequenza di calcolo per ogni mese

Ordine deterministico — nessuna equazione simultanea:

1. **Ricavi:** importo base × coefficiente stagionale × variazioni puntuali del mese
2. **Costi operativi:** applicazione driver (% su ricavi o importo fisso rivalutato)
3. **Oneri finanziari:** somma quote interessi di tutti i mutui attivi nel mese
4. **Ammortamenti:** somma quote pro-rata di tutti gli investimenti attivi
5. **Imposte:** aliquota su imponibile (IRES + IRAP con basi distinte)
6. **Utile netto:** risultato CE del mese
7. **Capitale circolante:** crediti/debiti/magazzino da DSO/DPO/DIO applicati a ricavi/costi mese
8. **Immobilizzazioni:** saldo precedente + Capex mese - Ammortamento mese
9. **Patrimonio netto:** saldo precedente + utile mese - dividendi (a fine anno)
10. **PFN:** variabile di chiusura — garantisce Totale Attivo = Totale Passivo + PN
11. **Cassa:** saldo banca = saldo precedente + flusso cassa operativo + eventi finanziari

### 4.3 Gestione della circolarità oneri finanziari / PFN

Gli oneri finanziari dipendono dal debito (PFN), che dipende dall'utile, che dipende dagli oneri finanziari. Si risolve con iterazione convergente:

- Iterazione 1: calcolo con oneri finanziari dell'anno precedente come stima iniziale
- Iterazioni 2-3: ricalcolo con PFN aggiornata
- Soglia di convergenza: variazione < 0,01% tra iterazioni successive

## 5. Gestione degli eventi puntuali

Ogni evento è un oggetto indipendente con una data e una logica propria. Si aggiunge, modifica e rimuove senza toccare il resto del modello.

### 5.1 Mutuo / finanziamento

```json
{
  "tipo": "mutuo",
  "descrizione": "Mutuo Banca X acquisto capannone",
  "data_inizio": "2025-03",
  "capitale_residuo": 280000,
  "tasso_annuo": 0.045,
  "durata_mesi": 120,
  "tipo_ammortamento": "francese"
}
```

### 5.2 Investimento in cespite

```json
{
  "tipo": "investimento",
  "descrizione": "Acquisto linea produzione",
  "data_acquisto": "2026-04",
  "importo": 150000,
  "categoria_sp": "B.II",
  "voce_sp": "B.II.2",
  "aliquota_ammortamento": 0.15,
  "metodo": "quote_costanti"
}
```

### 5.3 Variazione ricavi / costi

| Tipo | Descrizione |
|---|---|
| A — Puntuale | Variazione % su singolo mese/anno specifico |
| B — Strutturale | Variazione % del run-rate da un mese in avanti, permanente |
| C — Annuale | Crescita % da gennaio dell'anno — driver semplice |

---

## 6. Modello ricavi — profilo mensile

```
ricavo_mese(M, Y) =
  importo_annuale_base(Y)           # base storica + eventi strutturali fino a M/Y
  × profilo_stagionale[M]           # coefficiente mensile (somma 12 coeff. = 100%)
  × (1 + variazione_puntuale[M][Y]) # se presente, altrimenti 1.0
```

## 7. Interfaccia utente

### 7.1 Struttura generale

L'interfaccia è composta da:
- **Sidebar sinistra** — navigazione principale, sfondo #1B3A5C (blu scuro), testo bianco
- **Header** — titolo sezione corrente, badge cliente, badge scenario, pulsanti salva/annulla
- **Area contenuto** — pannello principale con tab, toolbar e tabella dati
- **Footer status bar** — stato salvataggio, quadratura SP, totali

### 7.2 Sezioni di navigazione (sidebar)

| Sezione | Funzione |
|---|---|
| Dati di partenza | SP e CE storici — tab separati, mastri collassabili |
| Driver & Parametri | Ricavi per voce, profili stagionali, DSO/DPO/DIO, fiscale |
| Eventi | Mutui, investimenti, variazioni strutturali con date precise |
| Prospetti futuri | SP e CE proiettati per anno — drill-down mensile disponibile |
| Dashboard KPI | Grafici e KPI principali — EBITDA, PFN, cassa, margini |

### 7.3 Sezione "Dati di partenza"

- Tab separati per SP e CE
- Toolbar con: anno di riferimento, pulsanti espandi/comprimi tutto, toggle Rapida/Analitica
- Barra scenario in cima: "SP + CE storici" / "Solo SP storico" / "Società costituenda"
- Mastri collassabili/espandibili con click sulla riga
- Sottomastri collassabili indipendentemente dai mastri
- In modalità Analitica: pulsante "Aggiungi conto" per ogni sottomastro
- Subtotali e totali calcolati automaticamente a ogni modifica
- Controllo quadratura SP nel footer (Attivo = Passivo + PN)

### 7.4 Regole UI critiche — da rispettare sempre

- **Nessun elemento form nativo** per input e pulsanti: usare `div` con `contenteditable`
  per i campi importo e `div` con `onclick` per i pulsanti. Gli elementi `input` e `button`
  nativi vengono sovrascritti dal renderer con sfondi scuri illeggibili.
- **Nessun inline style per colori:** tutti i colori definiti in classi CSS in `main.css`.
  Usare variabili CSS in `:root` per tutti i colori principali — facilita modifiche globali.
- **Palette colori:** sfondo bianco/grigio chiaro/azzurro chiarissimo. Testo sempre scuro
  su sfondo chiaro. Mai testo chiaro su sfondo chiaro.
- **Font testo:** DM Sans (Google Fonts)
- **Font importi numerici:** JetBrains Mono (Google Fonts), dimensione minima 14px
- **Sidebar:** sfondo #1B3A5C, testo bianco/semitrasparente, voce attiva con sfondo
  rgba(55,138,221,0.22)

### 7.5 Protezione dati e salvataggio

- Salvataggio manuale: pulsante "Salva" scarica il file JSON aggiornato
- Indicatore visivo di stato nel footer: `Pronto` / `Modifiche non salvate` / `Salvato`
- Blocco chiusura scheda browser: evento `beforeunload` con popup di conferma se ci sono modifiche non salvate
- Caricamento progetto: drag & drop del file JSON o pulsante "Apri progetto"
